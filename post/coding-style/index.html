关于码风

2020-02-26 2 min read # 关于lyc # はるまきごはん # 日常灌水

大括号

  • 大括号不换行
  • for/while只有一行不开大括号，换行
for (int i=1;i<=n;i++)
	cin>>a[i];

循环

  • 从1开始并喜欢用<=
  • i++而不是++i
  • 不让for循环中三栏参数有一栏空着（for (;i<=n;i++)），否则用while
  • 循环变量不在循环之前定义

缩进

  • 四格缩进
  • Tab而不是空格

常规

  • 运算符之间如果不必须加空格则不加空格（for/while/if例外）
  • 两部分代码/两个循环/两坨段定义之间空一行
  • 一个int/long long/.....只定义一个或一组变量/数组
  • 变量名大多比较常规，有时候会有奇怪的变量名蹦出来
  • cin/cout而不是scanf/printf
  • endl而不是\n
  • 有时候会加ios::sync_with_stdio(false);
  • 会写using namespace std;
  • 懒得写读优
  • 一般只写用到的头文件而不是#include <bits/stdc++.h>
  • int main()与大括号之间空一行（很sb对吧

像这样

#include <iostream>
#include <set>
.......
using namespace std;
int main()

{
	......
	return 0;
}
  • 函数写在main前面而不是后面
  • vector 而不是邻接表（因为lyc太菜了
  • true/false而不是1/0
  • STL而不是手写容器
  • 会写#define int long long
  • 会使用 C++11 特性（for (auto a:v) 这种东西）
